# ðŸŒ Internationalization & Local Assets (I18N)

SoftLicence supports multi-language management (currently **English** and **French**) and is designed to operate in restricted or industrial environments without Internet access.

## ðŸ› ï¸ Configuration

The server-side localization is handled by `IStringLocalizer` and resource files (`.resx`) located in the `Resources/` folder.

- **Default Language**: English (`en`).
- **Supported Languages**: English (`en`), French (`fr`).

To change the default language, modify the `Program.cs` file:
```csharp
app.UseRequestLocalization(options => {
    options.SetDefaultCulture("en"); // Change to "fr" for French
    // ...
});
```

## ðŸ“¦ Local Assets (Offline Mode)

All UI dependencies are hosted locally in `wwwroot/lib/` to ensure the administration console works without a CDN or Internet connection.

### Bootstrap
- **CSS**: `wwwroot/lib/bootstrap/css/bootstrap.min.css`
- **JS**: `wwwroot/lib/bootstrap/js/bootstrap.bundle.min.js`

### Flag Icons
Flags are managed via the **flag-icons** CSS library (loaded via CDN in `App.razor`). They are used in the language selector and the audit log.

## ðŸš© Adding a New Language

If you want to add a new language (e.g., Spanish - `es`):

### 1. Resource Files
1. Create a copy of `SharedResource.en.resx` and rename it to `SharedResource.es.resx`.
2. Translate all the `<value>` tags inside the XML.

### 2. Update the UI
In `src/SoftLicence.Server/Components/Layout/MainLayout.razor`, add the new language to the dropdown using the appropriate CSS class (`fi fi-es`):
```html
<li>
    <a class="dropdown-item d-flex align-items-center gap-2" href="/Culture/Set?culture=es&redirectUri=@System.Net.WebUtility.UrlEncode(Nav.Uri)">
        <span class="fi fi-es"></span> EspaÃ±ol
    </a>
</li>
```

### 4. Enable in Program.cs
Add the culture code to the supported cultures list in `Program.cs`:
```csharp
var supportedCultures = new[] { "en", "fr", "es" };
```
